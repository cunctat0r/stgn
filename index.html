Система мониторинга представляет собой телеметрическую систему, построенную в виде сети с одним терминальным узлом, подключенным к персональному компьютеру, и удаленными узлами сбора данных с датчиков. Система решает следующие задачи.

  • Удаленный мониторинг ВЛ для контроля гололедно-изморозевых отложений при котором поступающая информация с постов телеизмерений в диспетчерский пункт достаточна, своевременна и точна для принятия решения о времени начала и окончания плавки ВЛ, очерёдности проплавляемых ВЛ, направлении движения фронтов отложений.
  • Контроль особо важных опор (переходных, угловых, анкерных и т.д.) от случаев вандализма, хищения уголка, повреждение оттяжек на опорах, так же периодически происходящих наездов транспорта на опоры, со своевременным информированием диспетчера сетей о всех произошедших отклонениях и перераспределениях усилий.

Алгоритм работы системы. Каждый удаленный узел периодически выходит в сеть GSM и передает данные, собранные с датчиков. Терминальный узел постоянно находится в режиме приема и регистрирует данные по мере прихода их с удаленных объектов. Данные, приходящие с удаленных объектов, организованы в пакеты. Каждому временному интервалу регистрации соответствует два пакета по 60 байт каждый, которые передаются последовательно от 3 до 8 раз для повышения надежности. Дополнительно осуществляется проверка контрольной суммы каждого пакета. Пакеты передаются по протоколу UDP в случае использования технологии GPRS либо в двоичном виде при использовании технологии CSD.

Величины показаний датчиков, полученные после расшифровки посылки, полученной приемным модемом, помещаются в базу данных, реализованную в СУБД Microsoft Access либо MySQL. Отображение необходимых диспетчеру параметров обеспечивается клиентской программой либо с помощью веб-интерфейса через установленный на клиентском компьютере браузер. В последнем случае возможно размещение базы данных на сервере в сети интернет, что обеспечивает возможность доступа к результатам мониторинга в любой точке земного шара.
